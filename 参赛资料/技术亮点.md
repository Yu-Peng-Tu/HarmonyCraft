## 二、关键技术实现

### 2.1 AI思考链4步化（ChatPage核心）

**问题**：单次生成AI响应缺乏透明度和可感知性，用户无法了解生成过程。

**方案**：将生成过程拆解为4个可感知的步骤，每步推入thinking消息，完成后再移除。

```typescript
// 伪代码示意
async simulateAIResponse(userInput: string) {
  // Step 1: 理解需求
  this.pushThinking('正在理解您的需求...');
  await delay(600);
  this.popThinking();
  
  // Step 2: 分析布局
  this.pushThinking('分析页面结构...');
  await delay(500);
  this.popThinking();
  
  // Step 3: 生成组件
  this.pushThinking('生成ArkTS代码...');
  await delay(700);
  this.popThinking();
  
  // Step 4: 优化样式
  this.pushThinking('优化视觉效果...');
  await delay(400);
  this.popThinking();
  
  // 最终输出代码消息
  this.pushCodeMessage(generatedCode);
}
```

**效果**：用户可感知AI"正在工作"，减少等待焦虑，同时展示AI的专业性。

### 2.2 可交互预览渲染（PreviewRenderer）

**问题**：传统代码预览仅为静态截图，无法验证交互逻辑。

**方案**：为每个场景构建独立的`@Component`预览组件，使用真实的`@State`状态管理。

```typescript
// 以计数器为例
@Component struct CounterPreview {
  @State count: number = 0;
  
  build() {
    Column() {
      Text(this.count.toString())
      Button('增加').onClick(() => this.count++);
      Button('减少').onClick(() => this.count--);
    }
  }
}
```

**覆盖场景**：计数器（±）、登录（输入验证）、待办（勾选/删除）、设置（Toggle切换）、音乐（播放/暂停）、聊天（发送消息）、天气（数据展示）。

### 2.3 类型安全的路由参数传递

**问题**：ArkTS严格模式下`Record<K,V>`和索引签名`[key:string]`被禁止使用。

**方案**：为每个页面间传参定义显式接口，彻底规避类型不安全。

```typescript
// ChatPage.ets
interface ChatPageParams {
  prompt: string;
}

// CodePreview.ets
interface CodePreviewParams {
  code: string;
}

// ProjectDetail.ets
interface ProjectDetailParams {
  projectId: string;
  projectName: string;
  projectType: string;
  updateTime: string;
}
```

**参数规范**：
- Index→ChatPage: `{prompt: string}`
- ChatPage→CodePreview: `{code: string}`
- TemplateMarket→ChatPage: `{prompt: string}`
- ProjectManage→ProjectDetail: `{projectId, projectName, projectType, updateTime}`

### 2.4 AppStorage数据持久化闭环

**问题**：元服务/应用重启后数据丢失。

**方案**：所有用户数据通过`AppStorage`持久化，包括：

| 数据类型 | Key | 结构 |
|---------|-----|------|
| 已保存项目 | `saved_projects` | `ProjectItem[]` |
| 项目版本 | `project_versions` | `ProjectVersion[]` |
| 收藏模板 | `saved_templates` | `string[]` |
| 深色模式 | `dark_mode` | `boolean` |

```typescript
// Models.ets 核心API
saveProjectToStorage(name, type, code?): void
saveProjectVersion(projectId, code, note): void
getProjectVersions(projectId): ProjectVersion[]
getSavedProjects(): ProjectItem[]
deleteProject(id): void
renameProject(id, newName): void
```

### 2.5 暖色调品牌体系（Theme.ets）

**设计理念**：区别于科技产品千篇一律的蓝紫渐变，采用琥珀陶土色系，传递"手工创造"的温暖质感。

| 语义 | 色值 | 使用场景 |
|-----|------|---------|
| 主色 | `#D4763E` | 按钮、图标选中态、强调元素 |
|  accent | `#B85C3D` | 次级按钮、标签 |
| 主色浅 | `#E89860` | 悬停态、渐变端点 |
| 背景主 | `#FAF6F1` | 页面全局背景 |
| 卡片背景 | `#FFFDF9` | 卡片、浮层 |
| 文字主色 | `#2D2420` | 标题、正文 |
| 文字次色 | `#6B5D54` | 描述、辅助信息 |
| 代码背景 | `#2D2420` | 代码编辑器深色背景 |
| 代码编辑区 | `#1A1714` | 代码编辑区更深背景 |

### 2.6 纯矢量图标系统（CraftIcon）

**设计规范**：
- 统一1.75px描边宽度
- 圆角端点（StrokeLineCap.Round）
- 圆角连接（StrokeLineJoin.Round）
- 24×24视口

**覆盖图标**：home/templates/projects/profile/send/mic/search/back/close/add/check/more/login/todo/counter/news/settings/datalist/copy/download/phone/tablet/pc/tag/popular（共24个）

**技术实现**：纯`Shape`+`Path`代码绘制，无外部图片资源依赖，包体更小、渲染更快、任意缩放不模糊。

### 2.7 万能卡片服务（FormExtensionAbility）

**配置链路**：
1. `module.json5`注册`FormExtensionAbility`
2. `form_config.json`定义卡片规格（1×2/2×2/2×4）
3. `FormAbility.ets`实现生命周期回调
4. `FormCard.ets`渲染卡片UI

**关键代码**：
```typescript
// FormAbility.ets
export default class FormAbility extends FormExtensionAbility {
  onAddForm(want: Want): formBindingData.FormBindingData {
    const data: FormData = { title: 'AI卡片', desc: 'HarmonyCraft' };
    return formBindingData.createFormBindingData(data);
  }
}
```

### 2.8 转场动画体系

所有主要页面进入时采用统一的opacity淡入动画：

```typescript
@State pageOpacity: number = 0;

onPageShow() {
  this.pageOpacity = 1;
}

build() {
  Column() {
    // 页面内容
  }
  .opacity(this.pageOpacity)
  .animation({
    duration: 250,
    curve: Curve.EaseOut
  })
}
```

**已接入页面**：Index、ChatPage、CodePreview、TemplateMarket、ProjectManage

## 三、代码规范与质量

### 3.1 严格模式遵循

| 禁止项 | 替代方案 | 原因 |
|-------|---------|------|
| `Record<K,V>` | 显式接口定义 | 类型安全 |
| `[key: string]` | 显式接口定义 | 类型安全 |
| `...` 展开运算符 | 逐个属性赋值 | ArkTS严格模式兼容 |
| `FontWeight.SemiBold` | `FontWeight.Medium` | 枚举兼容性 |
| `getContext()` | 直接操作AppStorage | API兼容性 |
| `any` | 具体类型 | 类型安全 |
